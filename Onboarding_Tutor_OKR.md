## O3: 确保学习效果可验证

通过客观选择题测试 + 严格评价体系 + 追问上限机制，确保每次学习的结果可量化、可追溯。

| KR | 关键结果 / 衡量标准 | 实现方式 / 方案决策 | 状态 |
|:---|:---|:---|:---:|
| **KR3.1** | **每主题出 3-5 道选择题，覆盖三维度**<br>衡量：概念理解 + 流程判断 + 场景分析全覆盖 | 方案选型：选择题（✅采用） vs 填空题（❌评分不可靠） vs 开放题（❌不适合新人） vs 编程题（❌偏离目标）<br>选项设计有区分度，不能通过排除法猜对<br>出题规范详见 references/test-template.md | ✅ |
| **KR3.2** | **客观评价执行率 100%，鼓励语禁用**<br>衡量：禁止"很棒/很好/没关系/已经很不错了"等用语 | 全对 → 告知正确率，指出易混淆点（不夸）<br>部分对 → 先指错误，再分析偏差原因（不安慰）<br>全错 → 直接指出理解偏差根源，建议重新学习（不安慰）<br>替代用语：客观陈述事实 | ✅ |
| **KR3.3** | **追问上限不超过 3 轮**<br>衡量：同一知识点3轮后强制收束 | 第1轮 → 详细讲解 + 追问确认<br>第2轮 → 换角度讲解 + 再次确认<br>第3轮 → 简化答案 → 标记"待巩固" → 强制推进下一主题<br>例外：用户主动提出的新角度问题不计入上限 | ✅ |
