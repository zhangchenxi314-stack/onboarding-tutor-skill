# Table Format Guide（表格排版规范）

## 适用场景

凡涉及以下内容，必须使用表格：

- 接口参数说明
- 状态枚举与流转
- 配置项对比
- 多个对象的属性对比
- 方案对比/优劣势分析
- 错误码说明

## 基本规则

1. **表头必填**：每个表格必须有明确的列标题
2. **最少 3 列**：对比表格至少要有 3 列（对象 + 维度1 + 维度2）
3. **对齐**：中文列默认左对齐，数字列右对齐
4. **空值**：如果某项不适用，填 `—`（EM DASH）而非留空

## 表格类型与模板

### 类型 A：接口参数表

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| user_id | string | 是 | — | 用户唯一标识 |
| amount | int | 是 | — | 金额（分） |
| currency | string | 否 | "CNY" | 货币代码 |
| timeout_ms | int | 否 | 3000 | 超时时间 |

### 类型 B：状态枚举表

| 状态 | 含义 | 触发条件 | 可转换至 |
|------|------|----------|----------|
| INIT | 初始化 | 记录创建 | PENDING |
| PENDING | 待处理 | 参数校验通过 | PROCESSING, CANCELLED |
| PROCESSING | 处理中 | 开始执行 | SUCCESS, FAILED |
| SUCCESS | 成功 | 执行完成 | —（终态） |
| FAILED | 失败 | 异常/超时 | PENDING（重试） |
| CANCELLED | 已取消 | 用户/系统取消 | —（终态） |

### 类型 C：对比表

| 维度 | 方案 A：同步调用 | 方案 B：异步消息 | 方案 C：事件驱动 |
|------|-----------------|-----------------|-----------------|
| 实时性 | 高（毫秒级） | 中（秒级） | 低（分钟级） |
| 耦合度 | 强 | 弱 | 极弱 |
| 复杂度 | 低 | 中 | 高 |
| 适用场景 | 核心交易链路 | 非关键通知 | 跨系统数据同步 |

### 类型 D：对比表中带推荐（加粗推荐选项）

| 维度 | Redis | 本地缓存 | 说明 |
|------|-------|----------|------|
| 一致性 | **强（推荐）** | 弱 | Redis 集中管理，无多节点不一致问题 |
| 延迟 | 网络开销 ~1ms | **极低（推荐）** | 本地内存访问，0 网络开销 |
| 容量 | 受内存限制 | 受内存限制 | 两者都需要内存规划 |

## 排版规则

1. **列宽**：按内容自适应，最长单元格决定列宽
2. **数字**：大数字用千分位（1,000,000）或单位（100w）
3. **百分比**：统一用 `XX%` 格式
4. **true/false**：用中文 `是/否` 或 `支持/不支持`

## 注意事项

1. 不要用表格替代时序图（流程用图，数据用表）
2. 表格行数控制在 15 行以内，超过考虑拆分
3. 表格前后各留一个空行
4. 复杂嵌套关系不适合表格，改用缩进列表
